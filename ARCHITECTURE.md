# Project Structure & Architecture

## Overview

The AI Movie Insight Builder is a full-stack Next.js application that integrates three main services:

```
┌─────────────────────────────────────────────────────────────────┐\n│                    Frontend (React/Next.js)                     │\n│  ┌─────────────────────────────────────────────────────────┐   │\n│  │  - Movie Search UI                                      │   │\n│  │  - Results Display (Cards, Sentiment, Reviews)         │   │\n│  │  - Error Handling & Loading States                     │   │\n│  └─────────────────────────────────────────────────────────┘   │\n└─────────────────────────────────────────────────────────────────┘\n                               ↓\n┌─────────────────────────────────────────────────────────────────┐\n│                    Backend (Next.js API Routes)                 │\n│  ┌─────────────────────────────────────────────────────────┐   │\n│  │  /api/movie - Orchestrates all data fetching           │   │\n│  └─────────────────────────────────────────────────────────┘   │\n└─────────────────────────────────────────────────────────────────┘\n         ↓                ↓                    ↓\n    ┌─────────┐      ┌─────────┐         ┌──────────┐\n    │  OMDB   │      │  IMDb   │         │  OpenAI  │\n    │   API   │      │ Scraper │         │   API    │\n    │ (details)      │(reviews)│         │(sentiment)│\n    └─────────┘      └─────────┘         └──────────┘\n```\n\n## File Structure Breakdown\n\n### `/src/app`\nMain application entry point with Next.js App Router.\n\n- **`layout.tsx`** - Root layout component\n  - Sets metadata\n  - Configures fonts\n  - Wraps all pages\n\n- **`page.tsx`** - Home page (Main UI)\n  - Search input handling\n  - State management\n  - Results display\n  - Error handling\n\n- **`globals.css`** - Global styles\n  - Tailwind imports\n  - Custom animations\n  - Base styles\n\n- **`api/movie/route.ts`** - API endpoint\n  - Handles GET requests with `imdbId` parameter\n  - Orchestrates:\n    1. OMDB API call for movie details\n    2. IMDb scraping for reviews\n    3. OpenAI API call for sentiment analysis\n  - Returns combined response\n\n### `/src/components`\nReusable React components with TypeScript support.\n\n- **`SearchInput.tsx`** - Input component\n  - IMDb ID input field\n  - Search button\n  - Input validation\n  - Error display\n\n- **`MovieCard.tsx`** - Movie details display\n  - Poster image\n  - Title, year, rating\n  - Plot and cast info\n  - Responsive layout\n\n- **`SentimentDisplay.tsx`** - Sentiment analysis results\n  - Sentiment classification badge\n  - Summary text\n  - Visual indicator bar\n  - Emoji-based sentiment icons\n\n- **`ReviewsDisplay.tsx`** - Sample reviews\n  - List of audience reviews\n  - Truncated text display\n  - Review count indicator\n\n### `/src/lib`\nUtility functions and types.\n\n- **`utils.ts`** - Helper functions\n  - Type definitions\n  - IMDb ID validation\n  - Text formatting utilities\n  - Sentiment classification helper\n\n### Configuration Files\n\n- **`next.config.js`** - Next.js configuration\n  - Turbopack settings\n  - React strict mode\n\n- **`tailwind.config.mjs`** - Tailwind CSS configuration\n  - Custom animations\n  - Theme customization\n  - Content paths\n\n- **`postcss.config.mjs`** - PostCSS configuration\n  - Tailwind CSS plugin\n\n- **`tsconfig.json`** - TypeScript configuration\n  - Path aliases (`@/*`)\n  - Compiler options\n  - Strict type checking\n\n- **`jest.config.js`** - Jest testing configuration\n  - Test environment\n  - File extensions\n  - Ignore patterns\n\n- **`vercel.json`** - Vercel deployment configuration\n  - Build command\n  - Environment variables\n  - Node version\n\n- **`package.json`** - Project metadata\n  - Dependencies\n  - Scripts\n  - Version info\n\n## Data Flow\n\n### User Interaction\n\n```\n1. User enters IMDb ID (e.g., tt0133093)\n               ↓\n2. Frontend validates format (tt + digits)\n               ↓\n3. Click Search button or press Enter\n               ↓\n4. Frontend makes GET request to /api/movie?imdbId=tt0133093\n               ↓\n5. Backend processes request:\n   a) Fetch movie details from OMDB API\n   b) Scrape reviews from IMDb\n   c) Send reviews to OpenAI for sentiment analysis\n               ↓\n6. Backend returns combined response\n               ↓\n7. Frontend displays:\n   - Movie card with details\n   - Sentiment display with classification\n   - Sample reviews\n```\n\n## API Route Logic\n\n### `/api/movie` (route.ts)\n\n```typescript\nGET /api/movie?imdbId=tt0133093\n│\n├─ Validate imdbId\n├─ Fetch from OMDB: {\n│    Title, Year, Rated, Poster,\n│    Actors, Plot, imdbRating\n│ }\n├─ Fetch reviews from IMDb: [\n│    \"Great movie!\",\n│    \"Loved it!\",\n│    ...\n│ ]\n├─ Analyze sentiment with OpenAI: {\n│    summary: \"Mostly positive reviews...\",\n│    classification: \"positive\"\n│ }\n└─ Return: { details, reviews, ai }\n```\n\n## Component Hierarchy\n\n```\nRootLayout\n└─ page.tsx (Home)\n   ├─ SearchInput\n   ├─ Loading State\n   ├─ MovieCard\n   ├─ SentimentDisplay\n   └─ ReviewsDisplay\n```\n\n## State Management\n\nUsing React hooks in `page.tsx`:\n\n```typescript\nconst [id, setId] = useState(\"\");              // Input value\nconst [movie, setMovie] = useState(null);      // Movie details\nconst [reviews, setReviews] = useState([]);    // Reviews array\nconst [ai, setAi] = useState(null);            // Sentiment result\nconst [loading, setLoading] = useState(false); // Loading state\nconst [error, setError] = useState(null);      // Error message\n```\n\n## Error Handling\n\n1. **Frontend**: Input validation and error display\n2. **API**: Try-catch blocks with meaningful error messages\n3. **External APIs**: Graceful fallbacks and retries\n\n## Performance Optimizations\n\n1. **Image Optimization**: Next.js Image component\n2. **Code Splitting**: Automatic with Next.js\n3. **CSS**: Tailwind purges unused styles\n4. **Caching**: OMDB API responses cacheable\n5. **Lazy Loading**: Components load as needed\n\n## Type Safety\n\nFull TypeScript support with interfaces:\n- `MovieDetails` - Movie information\n- `SentimentResult` - AI analysis result\n- `MovieResponse` - API response structure\n\n## Styling Architecture\n\n- **Framework**: Tailwind CSS (utility-first)\n- **Custom CSS**: In `globals.css` and components\n- **Responsive**: Mobile-first design\n- **Animations**: CSS keyframes and Tailwind animations\n- **Dark Mode Ready**: CSS variables for theming\n\n## Testing Strategy\n\n- **Jest**: Unit testing framework\n- **ts-jest**: TypeScript support\n- **Basic Tests**: Input validation, API responses\n- **Coverage**: Expandable as project grows\n\n## Deployment Architecture\n\nOptimized for Vercel:\n- Serverless functions for API routes\n- Edge network for fast delivery\n- Automatic HTTPS\n- Environment variable management\n- GitHub integration\n\n## Technology Justification\n\n| Technology | Why Chosen | Benefits |\n|-----------|-----------|----------|\n| Next.js | Full-stack framework | SSR, API routes, optimizations |\n| React 19 | Modern UI library | Component reusability, hooks |\n| TypeScript | Type safety | Catch errors early, DX |\n| Tailwind CSS | Utility CSS | Fast development, responsive |\n| Axios | HTTP client | Promise-based, interceptors |\n| Cheerio | Web scraping | Lightweight, jQuery-like API |\n| OpenAI | AI analysis | SOTA sentiment analysis |\n| Vercel | Deployment | Best Next.js support |\n\n## Scalability Considerations\n\nFor future growth:\n1. **Caching Layer**: Redis for API responses\n2. **Database**: Store search history and favorites\n3. **Rate Limiting**: Protect API endpoints\n4. **CDN**: Serve static content globally\n5. **Analytics**: Track user behavior\n6. **Authentication**: User accounts and profiles\n\n## Security Features\n\n1. **Environment Variables**: Secrets not in code\n2. **Input Validation**: IMDb ID format check\n3. **CORS**: API protection\n4. **Error Messages**: No sensitive data exposure\n5. **HTTPS**: Automatic with Vercel\n
